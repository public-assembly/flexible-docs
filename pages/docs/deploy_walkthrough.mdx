### Create Git Repository

Create a name for your new project. Make your repository public by unchecking the checkbox.

### Configure Project

Provide the required environment variables. Beside each input description is an example, but you must provide your own unique values.

`NEXT_PUBLIC_PLATFORM_INDEX` Platform index created via a Flexible template, i.e. `3`
<br></br>
`NEXT_PUBLIC_TOKEN_ADDRESS` Ethereum address of your DAO's NFT, i.e. `0xd2e7684cf3e2511cc3b4538bb2885dc206583076`
<br></br>
`NEXT_PUBLIC_ALCHEMY_KEY` API key from your Alchemy dashboard. For detailed instructions on how to retrieve an Alchemy API Key, [click here](https://www.youtube.com/watch?v=tfggWxfG9o0).
<br></br>
`NEXT_PUBLIC_WEB3STORAGE_TOKEN` API key from your web3.storage dashboard. For detailed instructions on how to retrieve a web3.storage API Key, [click here](https://web3.storage/docs/how-tos/generate-api-token/)
<br></br>
`NEXT_PUBLIC_CHAIN_ID` Chain ID of the desired network for your app. Put `1` for Ethereum Mainnet and `5` for the Goerli Testnet.
<br></br>
`NEXT_PUBLIC_SITE_TITLE` Name of your DAO, i.e. `Public Assembly`
<br></br>
`NEXT_PUBLIC_SITE_DESCRIPTION` Short string of text for SEO purposes, i.e. `Create what's missing.`
<br></br>
`NEXT_PUBLIC_TWITTER_HANDLE` Twitter handle of your DAO, without the @ symbol. i.e. `pblcasmbly`. If you don't have one, supply an empty string, i.e. `""`
<br></br>
`NEXT_PUBLIC_WEBSITE_URL` Link to an alternative website for your DAO. If you don't have one, supply an empty string, i.e. `""`

### Deploy

With your environment variables configured, you're now all set to deploy your project to Vercel.

When your project finishes building, youâ€™ll be redirected to a success screen and prompted to continue to your Vercel dashboard. Here you will be able to view you website by clicking on the autogenerated link.

That's everything! If you have any trouble with this process, please [open an issue](https://github.com/public-assembly/flexible/issues/new) on GitHub.
